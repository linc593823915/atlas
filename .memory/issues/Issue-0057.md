Decision:
中文 cookbook 的 month / week / day 学习页必须转成可直接阅读的教材正文，而不是只给学习者概念标题和跳转链接。

RFC:
问题是现有中文学习页虽然已经补了索引、链接、评分和追问，但日页正文仍偏任务卡。学习者每天打开页面后，仍需要跳到其他文件或外部资料，才能理解当天真正要学什么。

目标是让每个学习层级都具备书本式说明：
- 月页解释本月主题、边界、主线能力和验收口径。
- 周页解释本周知识、Atlas 场景、工作顺序和常见误区。
- 日页解释当天概念、Atlas 落点、实践步骤、误区和检查点。

非目标：
- 不删除原有证据链接。
- 不重写英文底层台账。
- 不引入新的外部资料依赖。

Interface:
教材页新增稳定章节：
- 月页：`本月教材正文`
- 周页：`本周教材正文`
- 日页：`今日教材正文`

日页原 `今日阅读` 调整为 `今日延伸阅读与证据`，表达其用途是深化和留证，而不是替代当天正文。

Implementation:
批量扩写 `docs/cookbook-zh` 下的月、周、日 Markdown 文件。扩写内容基于每个文件已有主题、阶段、知识点、交付物、风险和 Atlas 落地检查生成，保持中文叙述和现有链接结构。

Unit Test:
需要验证：
- 12 个 month 页均包含 `本月教材正文`。
- 52 个 week 页均包含 `本周教材正文`。
- 所有 day 页均包含 `今日教材正文`。
- day 页不再保留 `## 今日阅读` 作为唯一阅读入口。
- `programops checkrefs` 通过。

Review:
审查重点是教材正文是否改变学习者体验：打开单日页面时，应能先读到概念解释、系统场景、执行步骤和检查点，再选择是否继续看证据链接。

Benchmark:
本 issue 不涉及性能路径，不需要 benchmark。

Documentation:
更新 cookbook 使用说明，明确日页正文是主阅读材料，链接是延伸证据。

Memory:
完成后记录一条可复用结论：中文 cookbook 的日级学习页应优先自包含，链接只作为证据和深化入口。

Decision:
中文 cookbook 增加“原子概念补充”，并把高频短词条与高频 RFC 类词条接成稳定落点。

Reason:
在大量高频提示被收口后，仍然有一类特别容易被误判成“我懂了”的短词：
- main
- config.Manager
- threat model
- review note

以及一类反复出现的高频产物词：
- Agent RFC
- Job Runner RFC
- MCP RFC
- Eval Loop RFC

这些词太短、太常见，如果没有稳定落点，学习者会在 day 层持续停留在“认得名字”而不是“能解释边界”。

Future:

减少 day 文档里的短词抽象项
继续把 day 提示接回稳定文档

# Program Control Center

Last updated: 2026-06-29

## Purpose

This document defines the control plane for the one-year AI agent architect program.

It ties together:

- roadmap commitments
- Atlas engineering backlog
- issue-card execution
- review cadence
- evidence capture
- monthly and quarterly reporting

## Operating Model

### Planning Layer

- `ROADMAP`
  Defines the 52-week and 365-day learning structure.
- `Month Docs`
  Define monthly objectives and assessment standards.
- `Atlas Backlog`
  Defines the engineering milestone for the month.

### Execution Layer

- `Issue Cards`
  Break backlog work into executable units.
- `Issue Status Board`
  Defines state transitions and ownership expectations.
- `Review Calendar`
  Defines when checkpoints happen.

### Evidence Layer

- `Evidence Registry`
  Defines what counts as proof.
- `Review Archive`
  Stores monthly and quarterly review records.

### Decision Layer

- `Promotion Rubric`
  Interprets performance across time.
- `Hiring Bar Packet`
  Interprets whether the body of work is externally competitive.

## Monthly Control Loop

1. Confirm month objective and backlog.
2. Break work into issue cards and set target weeks.
3. Track card state through the status model.
4. Collect evidence continuously instead of after the fact.
5. Close the month with OKR grading, mentor review, and mock interview.
6. Update memory and roll incomplete work forward explicitly.

## Quarterly Control Loop

1. Aggregate the three monthly review packets.
2. Build the replay pack.
3. Run architecture walkthrough and mock panel.
4. Calibrate promotion signal and next-quarter risks.
5. Update Atlas roadmap, backlog priorities, and evidence gaps.

## Required Artifacts Per Month

- monthly objective
- monthly backlog
- issue-card set
- at least one review packet
- evidence references
- updated docs or runbooks
- memory entry for stable decisions

## Failure Modes

- backlog exists but issue cards do not
- issue cards exist but have no owner or status
- review happens without evidence
- evidence exists but is not linked to a month or issue
- roadmap advances while Atlas does not

## Control Questions

1. What is the exact monthly objective?
2. Which issue cards are truly in progress?
3. Which evidence proves this month is real?
4. What is blocked, and who owns the unblock?
5. What part of Atlas changed because of this month?

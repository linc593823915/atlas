# Evidence Registry

Last updated: 2026-06-29

## Purpose

This registry defines the evidence types that count during monthly reviews, quarterly replays, promotion calibration, and hiring-bar evaluation.

## Evidence Types

### Code Evidence

- merged change or ready-to-review change
- diff summary
- module ownership impact

### Design Evidence

- RFC
- interface note
- architecture diagram
- trade-off record

### Reliability Evidence

- unit tests
- integration tests
- drill notes
- traces
- benchmark results
- rollback or recovery notes

### Documentation Evidence

- operator guide
- developer guide
- review summary
- runbook
- onboarding note

### Growth Evidence

- retrospective
- mentor notes
- mock interview notes
- memory entry

## Registry Rules

- Evidence must be linked to a specific issue, week, month, or quarter.
- Evidence without context should not be used for promotion or hiring arguments.
- One strong artifact is not enough; evidence should show consistency over time.

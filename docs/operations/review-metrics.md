# Review Metrics

Last updated: 2026-06-29

## Purpose

These metrics measure whether the learning system is progressing with engineering rigor rather than just accumulating documents.

## Monthly Metrics

### 1. Issue Completion Rate

- Formula:
  `done issues / planned issues`
- Target:
  `>= 80%`

### 2. Validation Coverage

- Formula:
  `issues with tests, traces, drills, or benchmarks / done issues`
- Target:
  `>= 80%`

### 3. Documentation Freshness

- Formula:
  `issues with updated docs or runbook notes / done issues`
- Target:
  `>= 70%`

### 4. Review Closure Rate

- Formula:
  `issues with recorded review evidence / done issues`
- Target:
  `100%`

### 5. Evidence Completeness

- Formula:
  `issues with at least one linked evidence artifact / done issues`
- Target:
  `100%`

### 6. Blocked Aging

- Formula:
  `average days that blocked issues remain blocked`
- Target:
  `<= 7 days`

## Quarterly Metrics

### 1. Monthly OKR Pass Rate

- Formula:
  `months graded Meets or Strong / months in quarter`
- Target:
  `3 / 3`

### 2. Atlas Milestone Completion

- Formula:
  `quarter milestones demonstrably shipped / quarter milestones planned`
- Target:
  `100%`

### 3. Reliability Evidence Density

- Formula:
  `count of test, drill, trace, or benchmark artifacts in quarter`
- Target:
  `increasing trend quarter over quarter`

### 4. Mock Interview Readiness

- Formula:
  `interview areas rated 3 or 4 / total interview areas`
- Target:
  `>= 75%`

## Interpretation Rules

- High completion with low evidence is not success.
- High documentation with low Atlas movement is not success.
- Strong interview answers with weak issue closure are not success.
- Strong metrics must reflect both learning and real system movement.

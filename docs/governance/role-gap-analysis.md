# Role Gap Analysis

Last updated: 2026-06-29

## Purpose

This document helps compare the learner’s current state with the target role: enterprise AI agent architect.

## Target Role Expectations

- designs platform boundaries
- reasons about tools, memory, orchestration, governance, and operations together
- ships systems with evidence, not guesswork
- communicates clearly across design, implementation, and review

## Gap Categories

### 1. Skill Gap

- example:
  can implement but cannot yet defend system design clearly

### 2. Execution Gap

- example:
  understands what to build but does not close artifacts consistently

### 3. Evidence Gap

- example:
  has good work but weak tests, weak traces, or weak benchmark proof

### 4. Communication Gap

- example:
  architecture may be sound, but docs and review notes are not reusable

### 5. Leadership Gap

- example:
  strong individual output, weak coaching, calibration, or influence

## Decision Rule

- do not treat a strong interview loop as sufficient if Atlas evidence is weak
- do not treat busy activity as sufficient if system movement is weak
- use the gap analysis to reduce the next most important risk, not to list every weakness

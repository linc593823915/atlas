# Annual Readiness Matrix

Last updated: 2026-06-29

## Purpose

This matrix defines what readiness should look like across the year.

## Dimensions

- Results
- Technical Judgment
- Code Health
- Reliability
- Communication
- Learning and Influence

## Quarter 1 Target

- Results:
  can finish scoped work and produce visible Atlas movement
- Technical Judgment:
  explains local trade-offs clearly
- Code Health:
  writes maintainable foundational code
- Reliability:
  adds tests and basic failure handling
- Communication:
  keeps RFCs and docs understandable
- Learning and Influence:
  runs retrospectives and accepts feedback well

## Quarter 2 Target

- Results:
  owns interfaces and protocol-level work
- Technical Judgment:
  designs tool and MCP boundaries
- Code Health:
  improves consistency across modules
- Reliability:
  thinks about approval, guardrails, and traceability
- Communication:
  explains cross-module designs coherently
- Learning and Influence:
  starts coaching through documentation and review

## Quarter 3 Target

- Results:
  ships stateful workflows and evaluation loops
- Technical Judgment:
  reasons about failure recovery and governance
- Code Health:
  protects long-term changeability
- Reliability:
  treats traces, drills, and regression gates as core design inputs
- Communication:
  writes usable governance and operational docs
- Learning and Influence:
  turns mistakes into reusable standards

## Quarter 4 Target

- Results:
  integrates the platform end to end
- Technical Judgment:
  makes architecture-level trade-offs credibly
- Code Health:
  preserves maintainability while integrating subsystems
- Reliability:
  reasons in terms of SLOs, rollback, benchmarking, and operability
- Communication:
  can defend the whole system under review
- Learning and Influence:
  shows staff-ready signs in architecture explanation and reusable artifacts

## Year-End Signal

- strong L5-like with selected L6-like signs
- not just feature completion, but platform ownership
- not just answers, but evidence-backed answers

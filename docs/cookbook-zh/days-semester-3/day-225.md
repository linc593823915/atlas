## 今日核心知识点

- metrics 要和用户或 operator 感知到的失败挂钩，而不是收集一堆 vanity counter。
- threshold 和 regression gate 需要 owner、升级路径和误报处理策略。
- 本周最大风险是：设了很多 gate，但没人信、没人懂，也没人据此做决定。
- 今天至少要把“release gate 规则”的验证口径写出来。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。

- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

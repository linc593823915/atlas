## 今日核心知识点

- prompt injection 防御首先要识别不可信指令从哪里进入系统。
- misuse tests 要尽量贴近真实攻击者会做的事，而不是只写几个玩具字符串。
- 今天要特别保护的不变量是：恶意场景被阻断的同时，等价 benign 场景也必须能被系统区分和放行。
- 如果 contract 写不清，本周最可能爆炸的点是：以为 prompt hygiene 已经等于治理，忽略了 tool、trace 和数据面的攻击面。

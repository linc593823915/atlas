## 今日核心知识点

- eval operations docs 要说明怎么跑 eval、怎么读失败、怎么安全更新 dataset。
- trace interpretation 要能区分模型失败、工具失败、策略失败和数据质量问题。
- 本周最大风险是：做了 dashboard 和 trace，但没有任何操作手册告诉别人怎么用。
- 今天至少要把“评估运维说明”的验证口径写出来。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

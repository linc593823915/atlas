## 今日核心知识点

- durable workflow 先是 state model 问题：run、step、checkpoint 和 terminal outcome 要先被命名。
- memory boundary 要回答哪些事实属于 workflow state，哪些事实属于外部长期记忆。
- 文档和 review 要服务于证据：workflow RFC、state model 和 checkpoint boundary。
- 接手者最容易踩到的坑通常就是：把长流程当成一个会 while 循环的普通请求，只在出问题时临时持久化。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。

- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

## 今日核心知识点

- metrics 要和用户或 operator 感知到的失败挂钩，而不是收集一堆 vanity counter。
- 门禁只有在可重复 workload 上运行才有意义，否则每次结果都不可比较。
- 文档和 review 要服务于证据：metric spec、threshold table 和 release policy。
- 接手者最容易踩到的坑通常就是：设了很多 gate，但没人信、没人懂，也没人据此做决定。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

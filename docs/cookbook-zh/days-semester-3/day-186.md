## 今日核心知识点

- pause/resume 语义要有确定性，否则 resume 以后就变成了另一条流程。
- memory boundary 要回答哪些事实属于 workflow state，哪些事实属于外部长期记忆。
- 今天最该盯住的风险仍然是：把长流程当成一个会 while 循环的普通请求，只在出问题时临时持久化。
- 扩展之后仍然不能破坏：一次 resume 必须能够从保存状态和确定性代码中被完整解释。

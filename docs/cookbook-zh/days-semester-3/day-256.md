## 今日核心知识点

- misuse tests 要尽量贴近真实攻击者会做的事，而不是只写几个玩具字符串。
- 好的防御证据既要证明恶意流量被拦住，也要证明正常流量没有被误伤。
- 今天最该盯住的风险仍然是：以为 prompt hygiene 已经等于治理，忽略了 tool、trace 和数据面的攻击面。
- 扩展之后仍然不能破坏：恶意场景被阻断的同时，等价 benign 场景也必须能被系统区分和放行。

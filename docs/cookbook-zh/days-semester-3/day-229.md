## 今天为什么学这个

- 今天要证明“指标、Regression Gate 与发布规则”不是表面可运行，而是真的可验证。没有证据，所有“已经做好了”的判断都不可靠。

## 今日阅读

- 阅读现有测试和验证布局，确认哪些失败仍然会静默漏过。
- 对照本周交付“release gate 规则”和证据要求“metric spec、threshold table 和 release policy。”补齐验证矩阵。

## 今日写作

- [写测试样例、drill 方案或验证矩阵](../daily-execution-playbook.md#write-validation-matrix)，[明确今天要证明什么、不证明什么](../daily-execution-playbook.md#criterion-proof-scope)。

## 今日动手

- [运行测试、drill、smoke check 或测量](../daily-execution-playbook.md#run-validation-or-smoke)，把“release gate 规则”[变成有证据的结论](../daily-execution-playbook.md#criterion-evidence-conclusion)。

## 今日核心知识点

- 本周必须留下的证据是：metric spec、threshold table 和 release policy。
- 今天验证最该覆盖的风险是：设了很多 gate，但没人信、没人懂，也没人据此做决定。
- 测试不能破坏的不变量是：一次 gate failure 必须能对应到清楚的发布或调查动作。
- 门禁只有在可重复 workload 上运行才有意义，否则每次结果都不可比较。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

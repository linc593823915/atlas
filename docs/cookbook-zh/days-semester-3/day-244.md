## 今日核心知识点

- threat model 必须识别资产、角色、入口和攻击路径，而且要针对 Agent 系统的真实行为。
- RFC 的价值在于把 threat 映射到 controls 和 tests，而不是写一份泛安全 memo。
- 文档和 review 要服务于证据：governance RFC、threat model 和 risk taxonomy。
- 接手者最容易踩到的坑通常就是：写了一个听起来很安全的文档，但没有把任何高风险动作真正拉进模型。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

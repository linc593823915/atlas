## 今日核心知识点

- checkpoint store 设计要处理原子性、版本和部分写入后的恢复问题。
- resume model 需要一个明确 cursor 或 decision point，告诉 runtime 应该从哪里继续。
- 实现阶段必须守住：加载一个 checkpoint 后，系统必须只能得到一个明确的下一步。
- 今天写的每一段代码都要服务于证据“graph model、checkpoint store 和 resume path。”。

## 今日核心知识点

- 多 Tool routing 需要清楚的选择策略，而不是注册多个工具后随缘调用。
- routing 逻辑必须可解释，否则 operator 无法理解为什么模型做出某次工具选择。
- 文档和 review 要服务于证据：routing path、fallback case 和 tool negative tests。
- 接手者最容易踩到的坑通常就是：工具越来越多，但没有任何可审计、可解释的选择逻辑。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。

- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

- [运行测试、drill、smoke check 或测量](../daily-execution-playbook.md#run-validation-or-smoke)，把“Agent 怎么调 Tool”[变成有证据的结论](../daily-execution-playbook.md#criterion-evidence-conclusion)。

## 今日核心知识点

- 本周必须留下的证据是：schema、validator path 和第一条 end-to-end tool call。
- 今天验证最该覆盖的风险是：把“差不多符合预期”的自然语言当成可靠数据直接使用。
- 测试不能破坏的不变量是：任何进入后续逻辑的结构化输出都不能绕过 schema 验证。
- tool 返回值需要统一规范化，否则后面的 routing、eval 和 audit 都会变脆。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

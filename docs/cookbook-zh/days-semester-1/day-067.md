## 今日核心知识点

- 第一条 tool path 应该把输入、tool result 和最终输出结构串成一个完整闭环。
- tool 返回值需要统一规范化，否则后面的 routing、eval 和 audit 都会变脆。
- 今天最该盯住的风险仍然是：把“差不多符合预期”的自然语言当成可靠数据直接使用。
- 扩展之后仍然不能破坏：任何进入后续逻辑的结构化输出都不能绕过 schema 验证。

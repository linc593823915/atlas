## 今日核心知识点

- structured output 是 LLM 文本和确定性代码之间的桥梁，没有 schema 就没有稳定接口。
- tool 返回值需要统一规范化，否则后面的 routing、eval 和 audit 都会变脆。
- 文档和 review 要服务于证据：schema、validator path 和第一条 end-to-end tool call。
- 接手者最容易踩到的坑通常就是：把“差不多符合预期”的自然语言当成可靠数据直接使用。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

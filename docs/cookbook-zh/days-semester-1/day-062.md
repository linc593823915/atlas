## 今日核心知识点

- Agent role 要回答模型负责什么、runtime 负责什么、tool 负责什么，而不是只写一段 prompt。
- 第一个 Agent RFC 应该同时描述 role、tool schema 和 runtime orchestration 的分工。
- 文档和 review 要服务于证据：Agent RFC、tool inventory 和 output contract。
- 接手者最容易踩到的坑通常就是：做出一个能聊的 Demo，却没有任何可复用的 contract。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

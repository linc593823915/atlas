# 第一学期：基础与第一个 Agent

最后更新：2026-06-30

## 这一学期要解决什么问题

很多人“会用大模型”，但不会做一个真正能演进的 Agent 系统。

第一学期就是解决这个问题。

你要先打通三件事：

1. 后端基础
2. 分布式执行基础
3. 第一个单 Agent 服务

## 这一学期结束时，你应该具备什么能力

你应该能够：

- 解释一个 Agent 服务为什么需要配置、日志、测试、运行时边界
- 解释为什么队列、重试、超时、幂等在 Agent 系统中仍然重要
- 做出一个最小但可靠的单 Agent 流程
- 把结果落在 Atlas，而不是只存在于临时代码里

## 学期结构

### 第 1 个月：后端基础与服务骨架

目标：

- 把 Atlas 建成一个干净的后端服务骨架
- 建立 `config`、`logger`、`runtime`、`db/cache` 的基本分层

对应阅读：

- [第一个月：后端基础与服务骨架](month-01-backend-foundation.md)

### 第 2 个月：后台任务与失败处理

目标：

- 让 Atlas 具备后台任务执行模型
- 学会处理 retry、timeout、dead-letter、idempotency

### 第 3 个月：第一个单 Agent 服务

目标：

- 用 Responses API 和 Tools 做出一个可工作的单 Agent 服务
- 让输出可验证，而不是“看起来差不多”

## 这一学期的正确学习方式

### 不要一开始就追求复杂

第一学期不应该追求：

- 多 Agent
- 长链编排
- 花哨前端
- 复杂自治

第一学期应该追求：

- 边界清晰
- 结构干净
- 日志可看
- 错误可控
- 代码可测

### 不要跳过工程卫生

如果你觉得：

- 配置太基础
- 日志太基础
- Makefile 太基础
- 单元测试太基础

那你后面一定会在更贵的地方付出代价。

## 学期完成标准

第一学期完成时，至少应该满足：

- Atlas 能稳定启动和停止
- 配置和日志入口统一
- 后台任务模型成立
- 有一个工具可调用的单 Agent 流程
- 有基本测试和文档

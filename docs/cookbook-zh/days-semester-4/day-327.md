## 今日核心知识点

- 本周必须留下的证据是：integration design、shared contract map 和 consistency check。
- 今天验证最该覆盖的风险是：内部有一套 tool truth，外部 MCP 又有一套 tool truth，长期必然分叉。
- 测试不能破坏的不变量是：同一个 capability 在 config、gateway 和 MCP 上只能有一份 canonical contract。
- 集成失败最常见的表现是命名不一致、schema drift 和 timeout 语义不一致。

## 今日核心知识点

- 配置传播要保证内部 runtime、外部 surface 和 tool 执行路径看到的是同一套 truth。
- MCP 暴露出的能力应与 gateway 强制执行的 schema、timeout 和 policy 保持一致。
- 实现阶段必须守住：同一个 capability 在 config、gateway 和 MCP 上只能有一份 canonical contract。
- 今天写的每一段代码都要服务于证据“integration design、shared contract map 和 consistency check。”。

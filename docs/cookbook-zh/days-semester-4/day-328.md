## 今日核心知识点

- config center、tool gateway 和 MCP integration 应该复用同一套 capability contract，而不是各自为政。
- 集成失败最常见的表现是命名不一致、schema drift 和 timeout 语义不一致。
- 文档和 review 要服务于证据：integration design、shared contract map 和 consistency check。
- 接手者最容易踩到的坑通常就是：内部有一套 tool truth，外部 MCP 又有一套 tool truth，长期必然分叉。

## 今日核心知识点

- config center、tool gateway 和 MCP integration 应该复用同一套 capability contract，而不是各自为政。
- MCP 暴露出的能力应与 gateway 强制执行的 schema、timeout 和 policy 保持一致。
- 今天要特别保护的不变量是：同一个 capability 在 config、gateway 和 MCP 上只能有一份 canonical contract。
- 如果 contract 写不清，本周最可能爆炸的点是：内部有一套 tool truth，外部 MCP 又有一套 tool truth，长期必然分叉。

## 今日核心知识点

- MCP surface 选择是 API product 决策：只暴露稳定、可消费的能力，而不是把内部接口直接倒出去。
- 第一版 server scope 必须刻意收窄，先学兼容性而不是一次性暴露全部能力。
- 文档和 review 要服务于证据：MCP RFC、capability inventory 和 surface scope note。
- 接手者最容易踩到的坑通常就是：把内部 helper 行为发布成长期协议 contract，后续无法演进。

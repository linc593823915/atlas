## 今日核心知识点

- gateway boundary 要区分用户意图、工具 contract 和工具实现，不能把所有细节都暴露给调用方。
- 统一入口的意义是让工具行为可治理、可观测、可测试，而不是只少写几行接线代码。
- 实现阶段必须守住：每一次 tool invocation 都必须走过同一条可检查、可拦截的 gateway path。
- 今天写的每一段代码都要服务于证据“Tool Gateway RFC、request/result envelope 和 boundary 说明。”。

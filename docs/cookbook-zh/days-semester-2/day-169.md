## 今日核心知识点

- orchestration tests 要覆盖多步委托、上下文传播和结果聚合，而不是只测单个 agent。
- fallback tests 应该模拟 child agent 卡住、失败或返回不可用结果时的退化行为。
- 本周最大风险是：demo 跑通一次，但没有任何可检查、可恢复、可追责的 runtime 证据。
- 今天至少要把“多 Agent 测试集”的验证口径写出来。

## 每日评分标准

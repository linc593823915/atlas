## 今日核心知识点

- orchestration tests 要覆盖多步委托、上下文传播和结果聚合，而不是只测单个 agent。
- runtime docs 需要说明如何检查一条 agent 链、如何理解审批流和回退路径。
- 今天要特别保护的不变量是：每一条 orchestrated run 都应该留下完整的决策链。
- 如果 contract 写不清，本周最可能爆炸的点是：demo 跑通一次，但没有任何可检查、可恢复、可追责的 runtime 证据。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。

- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

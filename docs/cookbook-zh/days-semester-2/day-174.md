## 今日核心知识点

- orchestration tests 要覆盖多步委托、上下文传播和结果聚合，而不是只测单个 agent。
- 月度复盘要证明多 Agent 复杂度换来了真实能力，而不是只换来更多 moving parts。
- 文档和 review 要服务于证据：orchestration tests、fallback tests 和 runtime docs。
- 接手者最容易踩到的坑通常就是：demo 跑通一次，但没有任何可检查、可恢复、可追责的 runtime 证据。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

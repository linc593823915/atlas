## 今天为什么学这个

- 对照本周交付“Tool Gateway 测试集”和证据要求“contract tests、negative tests 和 gateway review。”补齐验证矩阵。

## 今日动手

- [运行测试、drill、smoke check 或测量](../daily-execution-playbook.md#run-validation-or-smoke)，把“Tool Gateway 测试集”[变成有证据的结论](../daily-execution-playbook.md#criterion-evidence-conclusion)。

## 今日核心知识点

- 本周必须留下的证据是：contract tests、negative tests 和 gateway review。
- 今天验证最该覆盖的风险是：只测试工具能否成功调用，却从不验证 contract 被破坏时系统怎么表现。
- 测试不能破坏的不变量是：每一条 gateway promise 都应该至少有一个正例和一个负例。
- 月度复盘要说明 gateway 是否真的简化了工具接入，而不是只新增了一层抽象。

## 每日评分标准

- 满分 `20` 分。
- 阅读证据：5 分。
- 写作产物质量：5 分。
- 构建或实验推进：5 分。
- 输出清晰度与证据留存：5 分。
- 及格线：`12+`，且今天计划产物真实存在。
- 优秀线：`16+`，并能清楚说明取舍与下一步改进。

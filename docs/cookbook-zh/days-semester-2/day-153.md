## 今日核心知识点

- 多 Agent runtime 首先是 role map：谁负责规划、谁负责执行、谁负责 review、谁对最终结果负责。
- RFC 还要回答什么时候根本不该再起一个 agent。
- 文档和 review 要服务于证据：multi-agent runtime RFC、role map 和 handoff boundary。
- 接手者最容易踩到的坑通常就是：把多 Agent 设计成失控的层层委托，最终没人真正负责结果。

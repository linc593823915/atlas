## 今日核心知识点

- MCP surface 选择是 API product 决策：只暴露稳定、可消费的能力，而不是把内部接口直接倒出去。
- MCP RFC 要先定义 session lifecycle、transport 假设和 error shape，再谈实现。
- 今天要特别保护的不变量是：任何暴露到 MCP 的能力都必须脱离 Atlas 内部细节也能被解释和支持。
- 如果 contract 写不清，本周最可能爆炸的点是：把内部 helper 行为发布成长期协议 contract，后续无法演进。
